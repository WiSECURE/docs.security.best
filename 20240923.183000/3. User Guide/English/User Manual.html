User Manual

A. Welcome

Welcome to the WiSECURE CSE Solution! WiSECURE CSE enhances the client-side encryption capabilities of Google Workspace, offering an additional layer of security for your data. By integrating with the WiSECURE KACLS service, our solution ensures your files remain protected from unauthorized access while maintaining confidentiality and integrity. With WiSECURE CSE, you can enjoy complete control over your data security, even when it's stored in the cloud.

B. Security Description

WiSECURE CSE leverages advanced encryption technologies to provide robust client-side encryption (CSE) within Google Workspace. Through its integration with the WiSECURE KACLS service, the solution ensures comprehensive data protection using the following key components:

  • Compliance with Advanced Encryption Standards: Utilizing the AES-256 encryption algorithm, which meets the standards set by the National Institute of Standards and Technology (NIST), ensures that even if cloud data is intercepted, it cannot be decrypted.

  • Military-Grade Security Chip: Your encryption master key is safeguarded within a secure hardware module. This means that even if the hardware is compromised or tampered with, the key remains protected and cannot be stolen.

  • Portable and Lightweight Hardware: The Security Token, protected by a personal identification number (PIN), offers an intuitive and efficient security experience, making it easy to manage your encryption while ensuring maximum protection.

Even when encrypted data is stored in Google’s cloud or data centers, the use of end-to-end encryption means files are decrypted only within the user's browser. As a result, even Google cannot intercept or decrypt your files, giving you complete control over your data security.

Please note:

  • Applications on your computer (such as Chrome extensions) with sufficient permissions may be able to view or steal the content of decrypted files.

  • If someone can directly view your screen, encryption alone cannot prevent them from accessing the file's contents.

To use WiSECURE CSE solution, please ensure the following requirements are met:

  • You must have a Google Workspace account.

  • Your organization’s administrator must enable the WiSECURE CSE.

  • You must use the authorized Security Token provided by your organization and authenticate through the WiSECURE KACLS Agent.

B.2. WiSECURE KACLS Agent

WiSECURE KACLS is designed to enhance existing Google Workspace Client-Side Encryption (CSE). In traditional CSE services, the KACLS service operates in Google’s cloud, managing data key encryption and decryption. The revolutionary design of WiSECURE KACLS moves the KACLS service to the local endpoint and hands control over to the user’s Security Token.

Users can install the KACLS Agnet on their computer, verify their status as a legitimate holder of the Security Token, and securely communicate with Google servers. This enables users to securely use encrypted data across various Google Workspace services.

C. Instructions

Before using Google Workspace applications, ensure that the WiSECURE KACLS Agnet is running properly. (See: Start to use WiSECURE KACLS Agnet)

Please note that Google Workspace applications can only be used with Chrome or Edge browsers.

C.1. WiSECURE KACLS Agent

C.1.a. Start to use WiSECURE KACLS Agent

Prepare your initailized Security Token (see: Initialize Security Token - Blentity) and install the KACLS Agnet on your computer.

Follow these steps:

  1. Start the WiSECURE KACLS Agnet.

  2. Insert the Security Token into your computer’s USB slot.

  3. Enter your PIN.

  4. Enjoy the more secure Google Workspace!

    (Available for Chrome or Edge browsers only)

When using encryption and decryption, please ensure the KACLS Agnet remains active. Avoid closing the window or removing the Security Token, as this could interrupt the KACLS Agnet. For security reasons, the service will auto-stop after 2 hours of idle.

C.1.b. Change PIN

You can change your PIN directly through the WiSECURE KACLS Agnet. The steps are as follows:

  1. Start the KACLS Agnet.

  2. Insert the Security Token into your computer’s USB slot.

  3. Enter your PIN.

  4. Click the "Settings" icon next to "Logout" to open the settings interface.

  5. Select "Change PIN" from the left menu.

  6. Enter your current PIN and the new PIN, then click the "Change PIN" button to complete the process.

C.2. Google Drive

Your files in Google Drive, including Google Docs, Sheets, and Slides, will be encrypted and protected. WiSECURE KACLS ensures that only authorized users with the correct hardware key can decrypt to view or edit the files, preventing even Google from accessing them.

C.2.a. How to Create an Encrypted File

  1. Click "New" in the top left corner.

  2. Directly click the type of file you want to create to make an encrypted file. Alternatively, open the submenu and select "Blank encrypted document" to create an encrypted file.

The system will automatically encrypt and save your work every 30 seconds. You can click the cloud icon at the top, as shown in the image below, to save immediately. Alternatively, use the shortcut "Ctrl-S" or "Cmd-S."

Please note that automatic saving every 30 seconds will not occur if the device is idle or performing other actions, such as sharing or closing files.

Once the hardware key is removed or KACLS stops functioning, decryption and access to the content will not be possible.

C.2.b. Sharing Encrypted Files

You can share encrypted files following the standard process, but they can only be shared with individuals within the same organization. The recipient must use a valid Security Token to log in and decrypt the file.

Please note that only one person can edit an encrypted file at a time. For more details, refer to the guide on Collaborate on encrypted files.

C.3. Google Calendar

Your calendar events will be encrypted, including event descriptions, related meetings, and attachments. Only invited participants who have the correct hardware key can decrypt and access the events, even Google cannot read them.

C.3.a. How to Create an Encrypted Event

  1. Click "Create" in the top left corner.

  2. Choose “event“. The related content will be automatically encrypted by default, as shown in the image below.

Once the Security Token is removed or you log out of KACLS, you will no longer be able to see the content.

C.3.b. Creating an Encrypted Meeting via Calendar

You can directly create an encrypted video meeting link, which only invited participants within the same organization can join.

C.4. Google Meet

When using Meet, your video feed will be fully encrypted throughout the session. Only invited participants with a valid hardware key can join the meeting, and even Google cannot eavesdrop on or access the video stream.

C.4.a. Creating an Encrypted Meeting

You can schedule an encrypted meeting through Google Calendar or start one directly via Meet. Logging into Google Workspace, and follow these steps:

  1. Go to Google Meet and click "New meeting".

  2. Select "Video Call Options".

  3. Check the box for "Add encryption by (Your Organization)".

  4. Choose either “Create a meeting for later" or "Start an Instant Meeting".

C.4.b. How to Invite Others to an Encrypted Meeting

Encrypted meetings are restricted to members within the same organization. Participants must log into Google Workspace, activate KACLS, and pass Security Token verification.

Here’s how to invite others:

For Scheduled Encrypted Meetings

  1. Go to Google Meet and click "New meeting."

  2. Select "Video Call Options."

  3. Check the box for "Add encryption by (Your Organization)."

  4. Choose “Create a meeting for later".

  5. Copy the meeting information and send it to the members in the same Google Workspace organization.

For Instant Encrypted Meetings

  1. Go to Google Meet and click "New meeting."

  2. Select "Video Call Options."

  3. Check the box for "Add encryption by (Your Organization)."

  4. Choose “Starting an instant meeting”

  5. After enter the meeting, click the "Participants" icon in the menubar.

  6. Click "Add people."

  7. Choose the organization members you want to invite.

C.4.c. How to Join an Encrypted Video Meeting

Currently, encrypted meetings support only members of the same Google Workspace organization, and you must be invited to join. Follow these steps to join the meeting:

  1. Ensure the WiSECURE KACLS Agnet is running and verify your Security Token.

  2. Sign in with your Google Workspace credentials.

  3. Use one of the following methods to join the meeting:

    • Through a Google Calendar event

    • Via the meeting link URL

    • From an invitation received in Gmail

C.5. Gmail

With the WiSECURE CSE, email content is encrypted in the browser before being transmitted or stored in Google’s cloud storage. This adds an extra layer of protection for sensitive or regulated data, ensuring that all emails are securely protected before they reach the recipient's inbox.

Encryption Scope:

  • Email Content: Includes the body of the email, embedded images, and attachments, which are encrypted.

  • Email Headers: Includes the subject, timestamps, and recipients, which are not encrypted.

WiSECURE KACLS ensures that only authorized recipients with the proper hardware key can decrypt and read the email. Google cannot decrypt it.

Additionally, the Security Token can also be used to digitally sign Gmail, protecting them from tampering.

C.5.a. Sending Encrypted Emails

By default, you can send and receive encrypted emails within the same domain. For sending encrypted emails to external domains, refer to the instructions for sending encrypted emails to external recipients.

Important Notes:

Before you start drafting an email, decide whether it should be encrypted. By default, emails will be automatically encrypted. If you disable encryption while drafting, the draft will be deleted and a new draft will be opened.

Compose an Encrypted Email:

In Gmail, click "Compose" to start a new email. By default, the email will be encrypted. For encrypted mails, there will display a blue shield icon "message security" in the top-right corner of the email window.

To disable encryption, click the "message security" icon and click “Turn off”.

C.5.b. Sending Encrypted Emails to External Recipients

To send encrypted emails to recipients outside your domain, you must first exchange digital signatures. (See: How to Digitally Sign an Email)

Important Notes:

  • Emails with digital signatures will include your certificate and public key, which recipients can use to encrypt emails sent back to you.

  • Ensure that the recipient sends you the signed email back. Once the recipient sends the signed email, the system will automatically store the key, allowing for additional encryption in future communications with that recipient.

  • You only need to exchange digital signatures with each contact once.

  • If you or your contact updates your certificates, you will need to exchange digital signatures again.

C.5.c. How to Digitally Sign an Email

  1. In Gmail, click "Compose" to start a new email.

  2. Click the "Email Security" icon in the top-right corner of the email window.

  3. Select "Digital Signature."

  4. Check the box for "Sign massage."

To view and download your certificate, click “View signature."

C.6. Passwordless Authentication

Managing passwords can be costly, easy to forget, and prone to accidental exposure, creating significant security vulnerabilities for organizations. Advanced authentication technology allows users to log into multiple services, including Google, using a single Security Token, eliminating the need to remember multiple passwords.

C.6.a. Quick Login

Please ensure your Security Token is set up (see: Initialize Security Token - Blentity) and registered (see: Registering Security Token) with your Google Workspace account before attempting to log in.

  1. Go to the login page and enter or select your account.

  2. Follow the prompts to insert your Security Token.

  3. Input your PIN when prompted.

  4. Follow the instructions to touch the metal sensing areas on both sides of the hardware key.

  5. You will be logged in successfully.

C.6.b. Registering Security Token

  1. Go to the account management backend.

  2. In the left-hand menu, choose "Security."

  3. From the list of login methods, select "Passkeys and Security Keys."

  4. Click "Create a passkey."

  5. Click "Use another Device."

  6. Insert your Security Token and enter your PIN to verify.

  7. The binding process is complete

C.7. Initialize Security Token - Blentity

For first-time use of the organization-issued Blentity Security Token, please install the management tool to quickly set it up.

After initialization, you will setup a PIN and a passphrase.

C.7.a. Quick Setup

Please Ensure the switch on the end of the Blentity is turned on (showing orange).

  1. Insert the Blentity into a USB port on your computer.

  2. Press and hold the sensing areas on both sides of the Blentity for about 3 seconds to open the system disk.

  3. Click on the Blentity Manager program found on the disk to begin the installation.

Installation Path

  • For macOS: Blentity/Mac/Blentity Manager Installer.pkg

  • For Windows: Blentity/Windows/Blentity Manager Installer.exe

PIN

This is a security mechanism to protect the Security Token. You must enter the correct PIN to use the device. Therefore, even if the Security Token is lost or stolen, you don’t need to worry about it being misused by others.

Important: Do not share your PIN with anyone.

Passphrase

The passphrase represents the highest level of management control for your Security Token. It consists of 12 randomly generated English words and is used for (1) Secure Recovery of the Master Key (2) Unlocking the PIN (3) Restoring Backed-Up Files.

Important: Ensure that you memorize, copy, or write down the passphrase in order and store it securely. Do not share it with anyone.

D. Q&A

How Does WiSECURE KACLS Protect My Google Workspace Data?

WiSECURE KACLS is designed to enhance the existing Google Workspace Client-Side Encryption (CSE). While traditional CSE services operate within Google’s cloud, handling encryption and decryption of data keys, WiSECURE KACLS introduces a revolutionary approach by moving the KACLS service to the client side and placing control in the hands of the user with a Security Token.

By installing the KACLS Agnet on your computer, users can verify their ownership of the Security Token and communicate securely with Google servers, enabling secure use of encrypted data across various Google Workspace services.

How to Use ＷiSECURE KACLS Service

Launch the KACLS application, connect your Security Token to the computer, and then enter your PIN to activate the KACLS Agnet.

What Happens If I Forget to Stop the Service?

For security reasons and to minimize unnecessary resource usage, the KACLS Agnet will automatically stop after 2 hours of idle.

Can I Access Encrypted Data After KACLS Agnet Stops?

No, you cannot access encrypted files if the KACLS Agnet has stopped. You need to restart the KACLS program, insert the Security Token, and verify your PIN to regain access to encrypted files.

How Does WiSECURE’s Security Token Protect My Google Workspace Data?

WiSECURE’s Security Token provides the following key functions:

  1. Multi-Factor Authentication (MFA)

    The Security Token acts as a second authentication factor beyond passwords, supporting Google Workspace’s MFA and passwordless login. Even if your password is compromised, the Security Token and PIN ensure that neither Google, your organization’s administrators, nor anyone else can access your important data and emails. (For setup, refer to: Passwordless Authentication)

  2. Data Encryption Key Protection

    The Security Token performs hardware-level encryption (wrapping) and decryption (unwrapping) of data encryption keys in CSE, ensuring that only authorized users can decrypt data after verification.

  3. Digital Signatures

    The Security Token can provide hardware-based digital signatures for Gmail emails, ensuring that email content is not tampered with during transmission.

  4. Email Content Encryption

    The Security Token can encrypt Gmail email content, ensuring that even if data is exposed, unauthorized individuals cannot read it.

How to Safely Use or Share Data from Other Members in Google Workspace

WiSECURE CSE solution enhances the client-side encryption (CSE) capabilities of Google Workspace by integrating it with WiSECURE KACLS service, offering the highest level of hardware-based encryption. If your account belongs to the same Google Workspace group and has been properly enabled, you can directly select another member's account to share data, send emails, or invite them to meetings. The recipient must possess a Security Token and be authenticated by WiSECURE KACLS to decrypt and access the shared data.

For users in other domains, currently, only email sending and receiving is supported, and signature exchange is required. For more details, please refer to "Sending Encrypted Emails to External Recipients."

How to Use WiSECURE KACLS Service to Sign My Gmail Emails

  1. In Gmail, click “Compose” to start a new email.

  2. Click the “Message Security” icon in the top-right corner of the email window.

  3. Select “Digital Signature.”

  4. Check the box for “Sign message.”

  5. To view and download your certificate, click “View signature.”

How to Encrypt My Gmail Email Content

Email encryption must be set up at the time of creating the email. It is not possible to encrypt an existing email. (See: How to Create an Encrypted Email)

How to Identify Encrypted Files

Encrypted files will display a shield icon next to their file names.

Why Is a PIN Required When Using WiSECURE KACLS?

The PIN is used as a verification mechanism to prove that you are the legitimate owner of the Security Token, ensuring that others cannot misuse the key.
